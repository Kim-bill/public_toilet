## Component Inventory (Phase 1)

Primary focal point: the full-screen map canvas -- all supplemental UI (bottom sheet, loading overlay, error card) is layered over it and recedes when inactive.

### 1. Full-Screen Map Container

- Fills 100vw x 100vh (full viewport, no chrome)
- Kakao Map renders inside this container
- No app header or navigation bar in Phase 1 (map IS the app)
- z-index base: map at 0, bottom sheet at 10, loading overlay at 20

### 2. Toilet Markers

- Use Kakao Maps `kakao.maps.Marker` with custom `MarkerImage`
- Green circle marker for unlocked toilets (hasLock = false)
- Red circle marker for locked toilets (hasLock = true)
- Marker size: 32x32px (default Kakao marker size)
- Tap target: inherits from Kakao Maps (sufficient for mobile)

### 3. Bottom Sheet (toilet detail panel)

- **Trigger:** Tap any toilet marker (Decision D-02)
- **Position:** Anchored to bottom of viewport
- **Height:** Auto-height based on content, max 50vh
- **Layout (top to bottom):**
  1. Drag handle bar: 40px wide, 4px tall, centered, `border-radius: 2px`, color `#ccc`
  2. Building name: Heading role (20px bold)
  3. Lock status badge: Label role, colored badge (green "잠금 없음" / red "잠금 있음")
  4. Detail fields (each as label + value pair):
     - 층 (floor): e.g. "B1", "3F"
     - 위치 메모 (location memo): e.g. "푸드코트 옆 통로"
     - 비밀번호 (password): displayed in monospace, larger weight; show "없음" if empty
- **Animation:** Slide up from bottom, 200ms ease-out
- **Dismiss:** Tap outside bottom sheet, or swipe down on drag handle
- **Background:** White with `border-radius: 16px 16px 0 0` on top corners
- **Shadow:** `0 -2px 16px rgba(0, 0, 0, 0.1)`
- **Backdrop:** Semi-transparent overlay `rgba(0, 0, 0, 0.3)` over map

### 4. Loading State

- **When:** App startup while fetching toilet data from GAS (1-3 seconds typical)
- **Visual:** Centered spinner (CSS-only, 32px diameter) + text "화장실 정보를 불러오는 중..."
- **Position:** Overlay on top of map (map tiles may load before data)
- **Background:** `rgba(255, 255, 255, 0.85)` full-screen overlay
- **Dismiss:** Auto-dismiss when data fetch completes

### 5. Error State

- **When:** GAS fetch fails (network error, GAS timeout)
- **Visual:** Centered message card on map
- **Content:** See Copywriting Contract below

## Interaction Contracts

### Marker Tap

1. User taps a marker on the map
2. Map pans to center the tapped marker (Kakao Maps default behavior)
3. Bottom sheet slides up from bottom (200ms ease-out)
4. Bottom sheet displays toilet detail for tapped marker
5. If another bottom sheet is already open, it closes first (no stacking)

### Bottom Sheet Dismiss

1. User taps the backdrop overlay outside the bottom sheet
2. OR user swipes down on the drag handle
3. Bottom sheet slides down (150ms ease-in)
4. Backdrop fades out
5. Map becomes fully interactive again

### App Startup Sequence

1. HTML loads, Kakao Maps SDK initializes
2. Map renders centered on Misa New Town (37.5600, 127.1800) at zoom level 15
3. Loading overlay appears ("화장실 정보를 불러오는 중...")
4. Fetch toilet data from GAS endpoint (GET)
5. On success: create markers on map, dismiss loading overlay
6. On error: show error state with retry option

| 480-768px | Same as above, bottom sheet max-width 480px centered |
| > 768px | Not a priority target; map full-screen, bottom sheet max-width 480px centered at bottom |

Primary design target is mobile (< 480px). This is a phone-in-hand, walking-around app.

## Accessibility

| Concern | Approach |
|---------|----------|
| Touch targets | All interactive elements minimum 44x44px (bottom sheet drag handle area is 48px tall) |
| Color contrast | Password text (#2c3e50) on white background: ratio 10.7:1 (AAA). Lock badges use dark text on light tinted backgrounds. |
| Marker distinction | Locked (red) vs unlocked (green) markers are also distinguishable by users who cannot see color -- Phase 3 will add icon shapes. For Phase 1, color-only is acceptable for 2-user private app. |
| Screen reader | Bottom sheet content uses semantic HTML (`<h2>` for building name, `<dl>` for detail pairs). Pico CSS styles these natively. |
| Motion | Bottom sheet animation respects `prefers-reduced-motion: reduce` -- skip slide animation, show/hide instantly. |
