# Quick Plan 260410-k75: Search Dropdown with Distance and Toast Summary

Search results now display as a dropdown list (max 5) with GPS-based distance per item, replacing the previous first-result-only behavior and alert() feedback.

## Task Results

| Task | Name | Commit | Files |
|------|------|--------|-------|
| 1 | Dropdown container + styles + showToast export | e2484c1 | index.html, src/css/style.css, src/js/ui.js |
| 2 | Search results dropdown logic + distance + toast | e7322bc | src/main.js |

## What Changed

1. **index.html**: Added `<ul id="search-results">` dropdown container below search-bar
2. **src/css/style.css**: Added `.search-results` dropdown styling with place-name (ellipsis truncation) and place-distance (gray, right-aligned) styles; touch feedback on `:active`
3. **src/js/ui.js**: Added `export` keyword to `showToast` function for cross-module use
4. **src/main.js**: 
   - Imported `haversineDistance`, `formatDistance` from geo.js and `showToast` from ui.js
   - New `showSearchResults()` renders up to 5 results with distance labels
   - New `hideSearchResults()` clears and hides dropdown
   - `searchPlace()` now async -- fetches GPS position (graceful fallback), shows dropdown instead of auto-selecting first result
   - Replaced `alert('...')` with `showToast('...')` for no-results case
   - Added document click listener to close dropdown on outside tap
   - Clear button now also hides dropdown

## Decisions Made

- **showToast exported**: Reused existing toast implementation rather than creating a new one in main.js
- **Per-search GPS fetch**: Each search call attempts `getCurrentPosition()` for fresh distance data; if denied or failed, distance column is simply omitted (no error shown)
- **GPS alerts in GPS handler untouched**: Only search-related alert replaced with toast; GPS button alerts are out of scope for this plan
